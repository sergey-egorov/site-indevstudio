Проблема

Заказчик, телевещательная компания, обратился к нам с просьбой разработать приложение, которое бы значительно облегчило и ускорило работу по созданию субтитров для транслируемого видеоряда. На момент обращения создание субтитров проводилось вручную без использования специализированного программного обеспечения, что создавало значительную задержку по времени выхода телепередачи в эфир от видеопотока, генерируемого в режиме реального времени. Пожеланием заказчика было уменьшить это отставание, задействуя при этом минимально возможное количество технических и человеческих ресурсов.

Задачи

Нашей задачей стала разработка кросс-платформенного приложения для автоматической генерации субтитров, которое способно получать видеоряд, извлекать и распознавать речь, переводить звуковую дорожку в текстовую форму с сохранением и отображением временных меток, обеспечить автоматический перевод субтитров на другой язык и отображать созданные субтитры в транслируемом видеоряде. Программа должна обладать удобным, понятным интерфейсом для внесения правок (которых, увы, в любом случае избежать нельзя) в субтитры.

Решение

Проанализировав требования, мы пришли к решению задействовать облачный сервис, созданный нами для одного из предыдущих проектов, для задач по распознаванию речи и автоматическому переводу на другой язык. На основе Spring Boot и Java мы создали автономный сервер для обеспечения получения видеопотока, обработки звука и генерации текста.
После длительного обсуждения с заказчиком того, как должен быть устроен пользовательский интерфейс программы, мы довольно быстро и эффективно собрали его с использованием платформы Angular. Для того, чтобы работа по редактированию субтитров занимала как можно меньше времени, мы добавили набор горячих клавиш, дублирующих основные функции графического интерфейса.
